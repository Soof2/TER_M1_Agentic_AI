DECOMPOSITION EN SOUS-AGENTS (partie 5 uniquement)
---------------------------------------------------

Volonte explicite de l'utilisateur : pas d'over-engineering. On reste
a 3 sous-agents au lieu d'un par fichier. Chacun rend une section
Markdown autonome qui sera ensuite relue et integree manuellement.

SOUS-AGENT A — Coeur LangGraph
  Perimetre : graph.py, state.py, agents/A1-A8, prompts.py
  Choix a justifier :
    - StateGraph vs MessageGraph
    - Pattern Send / Reduce (Map/Reduce sur les evaluations)
    - interrupt_before pour HITL natif
    - Routage conditionnel : score > seuil -> A7 / sinon rapport
    - Separation A4 (Evaluateur) et A5 (Verificateur)
      = pattern de validation pair-a-pair
    - Choix Ollama local (souverainete, demo offline, pas de cout API)

SOUS-AGENT B — Outils & RAG
  Perimetre : tools/search.py, tools/rag.py, A3a/b/c
              (Stratege / Collecteur / Filtre)
  Choix a justifier :
    - Sources gratuites uniquement : DuckDuckGo + GitHub Search
      + Stack Exchange API
    - Filtre anti-bruit algorithmique (vs LLM-based)
    - Embeddings : SentenceTransformers all-MiniLM-L6-v2
    - Stockage : ChromaDB (local, pas de service externe)
    - RAG calibre par fiche de poste (deux collections + fiche_id)
      -> deja redige en detail dans rapportPlan.md
    - Seuil de calibration 0.5 : compromis sensibilite / utilite

SOUS-AGENT C — Service & Ops
  Perimetre : api.py, ui/, observabilite.py, docker-compose.yml,
              tests/
  Choix a justifier :
    - API Gateway pattern (UI = client comme un autre)
    - SSE vs WebSocket : SSE suffisant car flux unidirectionnel
    - Threading + queue.Queue : pont thread-safe pipeline -> SSE
    - NiceGUI vs Streamlit / Gradio / React
      -> deja redige en detail dans rapportPlan.md
    - Couverture des tests (81 tests, periemetre des assertions)
    - Observabilite : exports JSON timestampes
    - Docker compose : 3 services (ollama / api / ui)


CHOIX TRANSVERSAUX A ARGUMENTER (rediges par moi, pas par sous-agent)
---------------------------------------------------------------------

Choix qui touchent plusieurs parties du systeme et qui meritent un
traitement coherent au lieu d'etre eparpilles :

  - LangGraph plutot que LangChain / AutoGen / CrewAI
  - Ollama local plutot que API Anthropic / OpenAI
  - ChromaDB plutot que FAISS / Qdrant / Weaviate
  - Python pur cote frontend (NiceGUI) plutot que JS
  - API Gateway pattern plutot que UI couplee au graphe
  - HITL natif via interrupt_before plutot que mecanisme custom
  - Sources gratuites uniquement (impact RGPD + cout)
  - Pas de scraping LinkedIn (ToS, risque ban, hors perimetre TER)
